## バグ報告

バグを発見した場合は、GitHub Issuesで報告してください。以下の情報を含めていただけると助かります：

1. **バグの概要**: 何が起きたかを簡潔に。
2. **再現手順**: バグを再現するための具体的なステップ。
3. **期待される動作**: 本来どうあるべきだったか。
4. **実際の動作**: 実際に何が起きたか。
5. **環境情報**: OS、Node.jsのバージョンなど。
6. **スクリーンショット**:（もしあれば）

## 機能リクエスト

新しい機能のアイデアがある場合は、Issueを作成し、`enhancement` ラベルを付けてください。
どのような機能で、なぜそれが必要なのか、どのように動作すべきかを詳しく説明してください。

## プルリクエスト

1. リポジトリをフォークします。
2. 新しいブランチを作成します (`git checkout -b feature/amazing-feature`)。
3.変更をコミットします (`git commit -m 'Add some amazing feature'`)。
4. ブランチにプッシュします (`git push origin feature/amazing-feature`)。
5. Pull Requestを作成します。

**注意点**:
- 既存のコードスタイル（TypeScript, Prettier）に従ってください。
- 可能な限りテストを追加してください。
- PRの概要には、変更内容とその理由を明確に記述してください。

## 開発環境のセットアップ

README.mdの「インストール」セクションを参照してください。

```bash
git clone https://github.com/84kb/Obscura.git
cd Obscura
npm install
npm run dev
```
