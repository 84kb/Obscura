# コントリビューティング

Obscura プロジェクトへの貢献に関心を持っていただきありがとうございます。現在は開発中のアプリケーションであるため、変更の方向性や実装方針が大きく変わる可能性があります。大きな変更を始める前に、Issue での相談や提案の共有、あるいは既存の議論の確認をお願いします。

## バグ報告

本アプリはローカルメディア管理を前提としているため、環境差分やデータ構成によって問題の再現条件が変わることがあります。再現性のある不具合報告のため、GitHub Issues には以下の情報をできるだけ含めてください。

1. バグの概要
2. 発生手順
3. 期待される動作
4. 実際の動作
5. 使用環境
6. スクリーンショット

## 機能リクエスト

新しい機能のアイデアがある場合は、Issue を作成し、`enhancement` ラベルの利用を検討してください。なぜその機能が必要なのか、どのようなユースケースを想定しているのかが分かると、判断しやすくなります。

## プルリクエストの手順

1. リポジトリをフォークします。
2. 新しい作業用ブランチを作成します。`git checkout -b feature/amazing-feature`
3. 変更をコミットします。`git commit -m "Add some amazing feature"`
4. ブランチを push します。`git push origin feature/amazing-feature`
5. GitHub 上で Pull Request を作成します。

注意事項:

- 既存のコードスタイルや TypeScript / ESLint の方針に合わせてください。
- PR の説明には、変更内容とその意図が分かる説明を含めてください。

## 開発環境のセットアップ

README のインストール手順に従って環境を構築してください。

```bash
git clone https://github.com/84kb/Obscura.git
cd Obscura
npm install
npm run tauri:dev
```

改善提案があれば歓迎します。
