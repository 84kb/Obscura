# Obscura デザインチーム成果物（Round）
更新日: 2026-03-05  
担当: デザインチーム Pixel

## 1. 成果物サマリー（本チェックリスト 2）
本成果物は、以下3点のUI/UX仕様を確定し、開発・QAへ引き渡すための実装基準である。
- ライブラリカードの `dominant_color` 表示仕様
- `.library` サフィックス非表示の全画面統一仕様
- OSネイティブウィンドウ置換用のアプリ内モーダル/ダイアログ仕様

## 2. `dominant_color` 適用仕様
### 2.1 対象画面
- グリッド表示カード（`MediaCard`）
- リスト表示サムネイル枠（`LibraryList`）
- サムネイル未生成時のプレースホルダ領域

### 2.2 ルール
- 優先値:
  1. `dominant_color`
  2. `dominantColor`
  3. フォールバック色
- フォールバックは「値未取得/無効時のみ」適用する。
- 同一メディアに対して、グリッドとリストで色が不一致にならないことを必須にする。

### 2.3 視認性ガイド
- 背景色上の文字・アイコンはコントラスト比 4.5:1 以上を目標。
- サムネイル読込前の背景塗りは 200ms 以内に表示し、読み込み後にフェード遷移（既存アニメーション準拠）。
- 極端に明るい色はオーバーレイ（10-18%）で視認性を担保する。

### 2.4 受け入れ基準
- 20件サンプルで `metadata.json` の色値とUI表示が 100% 一致。
- 欠損データのみグレー表示になる（誤フォールバック 0件）。

## 3. `.library` 非表示ルール（全画面統一）
### 3.1 正規化ルール
- 末尾 `.library`（大小文字無視）のみ除去して表示名を生成。
- パス保存・内部IDは変更しない。
- 表示名が空文字化する異常ケースでは、元のベース名を表示。

### 3.2 適用面（最低保証）
- サイドバーの現在ライブラリ名
- サイドバーのライブラリ選択ドロップダウン
- 監査ログモーダル内のライブラリ名バッジ
- 設定画面内のライブラリ選択表示
- 通知/トーストでライブラリ名を表示する箇所

### 3.3 NG例
- `MyWork.library` がUI上でそのまま表示される
- 画面ごとに `MyWork` / `MyWork.library` が混在する

## 4. アプリ内モーダル/ダイアログ UI ガイド
### 4.1 種別
- `Info`: 完了通知、軽微な案内
- `Confirm`: 実行可否の確認（デフォルト）
- `Destructive`: 削除・取り消し不可操作
- `Input Prompt`: 文字入力を伴う確認

### 4.2 共通レイアウト
- Overlay: 背景暗転 + blur、クリック外側で閉じる可否は種別ごとに制御
- Header: タイトル（1行）、必要ならステータス色インジケータ
- Body: 本文、補足説明、改行保持
- Footer: 右寄せアクション（Cancel -> Primary/Destructive）

### 4.3 操作仕様
- `Esc`: キャンセル相当（取消不可モーダルは無効化）
- `Enter`: 主要アクション
- フォーカス初期位置:
  - Confirm系はキャンセルボタン
  - Destructive系はキャンセルボタン
  - Input Prompt系は入力フィールド
- フォーカストラップを必須化し、背景要素へTab遷移させない。

### 4.4 文言仕様
- タイトル: 30文字以内、操作意図を明示
- 本文: 80文字以内を基本、詳細は2段目へ分離
- 破壊的操作は取り消し不能を必ず明記

### 4.5 段階的置換優先度（UI観点）
- P0: ライブラリ削除、ファイル完全削除、リモート削除
- P1: 同期/更新失敗通知、設定保存エラー
- P2: 補助通知（コピー完了、軽微アラート）

## 5. 実装ハンドオフ要点（開発向け）
- 既存 `ConfirmModal` / `.app-modal` のトークン（`--bg-card`, `--border`, `--radius-*`）を共通化。
- メッセージボックス呼び出しを単一APIに寄せ、種別をオプションで切り替える。
- デザイン判断を伴う差分は、このドキュメントのルールを優先する。

## 6. QA連携チェック項目
- 色一致検証: `docs/templates/library_color_name_validation_20samples.csv`
- 画面統一検証:
  - 一覧、詳細、設定、監査ログで `.library` 非表示を確認
  - ライブラリ名の表示揺れ 0件
- モーダル置換検証:
  - P0対象でOSネイティブ呼び出し 0件
  - Enter/Esc/Tab の操作が仕様どおり

## 7. この成果物で確定したデザイン判断
- `dominant_color` は「取得できる限り表示」、グレーは例外時のみに限定。
- `.library` はユーザーに見せない内部実装情報として扱う。
- ネイティブダイアログ依存を減らし、アプリ内で一貫した体験に統一する。
